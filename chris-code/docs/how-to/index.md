# How-to guides

Task-oriented recipes. Each assumes you've done the [tutorial](../tutorials/getting-started.md) and know the pipeline — they show how to enter it for a specific job without starting from a blank design.

- **[Remediate a bug](remediate-a-bug.md)** — fix a known defect with a coherent, well-defended fix and a regression test.
- **[Make a determined change](make-a-determined-change.md)** — a refactor, migration, or API alignment where the behavior is settled and only the implementation is open.
- **[Review a branch](review-a-branch.md)** — get a senior-level review of work in progress, outside the automated gates.
- **[Run a quality campaign](run-a-quality-campaign.md)** — hunt for bugs, sweep the API surface with tests, or surface unfinished work.
- **[Cut a release](cut-a-release.md)** — bump the version, generate a changelog, and publish a GitHub release.

For the full list of skills and what triggers them, see the [Skills reference](../reference/skills.md).
