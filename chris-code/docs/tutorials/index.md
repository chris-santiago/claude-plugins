# Tutorials

Learning-oriented walkthroughs. Start here if you're new to chris-code — these teach the workflow by taking a real change from start to finish, with every step producing visible output.

- **[Getting started](getting-started.md)** — enable the plugin and drive one change through the full pipeline: brainstorm → spec → plan → execute → verify → finish.

Once you've completed the tutorial, the [how-to guides](../how-to/index.md) cover specific tasks, and the [explanation](../explanation/index.md) section covers *why* the workflow is shaped this way.
